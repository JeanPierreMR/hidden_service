About Tor

What is Tor hidden service?

Tor hidden service allow users to publish their service without revealing their identity (IP address).

Users can connect to this service using rendezvous point without knowing the publisher of service and revealing their identities.

This type of anonymity provides protection against distributed DoS attack as attacker would not know the IP address of service.

A rendezvous is a meeting, often a secret one, that you have arranged with someone for a particular time and place.

"Denial of service" or "DoS" describes the ultimate goal of a class of cyber attacks designed to render a service inaccessible.

Hidden service

What is a hidden service and how does it work?

When using the Internet, secure communication protocols make it possible to hide the data that a user exchanges with the server of a site. However, the fact that such communication takes place is usually visible. In other words, the content of the requests is hidden, but not the fact that they have been made.

A hidden service is an internet communication in which the web server is also anonymous. To understand what a hidden service is, it is necessary to know how onion routing works.

Onion Routing

How onion routing works

Onion routing is an anonymous communication system based on routing requests to a web server through a three-node system: Input node, Middle node and Output node.

Why Onion Services?

Onion services offer various privacy and security benefits to their users. An Onion Service's IP address is protected. Onion services are an overlay network on top of TCP/IP, so in some sense IP addresses are not even meaningful to Onion Services: they are not even used in the protocol.
